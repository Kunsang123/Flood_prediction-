# Flood_prediction-
This project is an end-to-end machine learning pipeline for predicting flood risk based on environmental, infrastructural, and socio-political factors. It is built using Python, Apache Airflow, MLflow, Docker, FastAPI, and Streamlit, and covers the full lifecycle from data ingestion and validation to model training, deployment, and monitoring.
Data is ingested and passed through three stages — raw, staging, and processed — with quality checks handled by Great Expectations. Three models are trained: Random Forest, XGBoost, and MLP, with Optuna managing hyperparameter tuning and MLflow tracking all experiments. The best model is deployed through a FastAPI REST API and visualized via a Streamlit dashboard. Prometheus and Grafana handle real-time monitoring, and data drift is detected automatically using the Kolmogorov-Smirnov test.
To run the project, Docker and Docker Compose need to be installed. Once cloned, all services can be started with a single Docker Compose command. The pipeline is then triggered through the Airflow UI at http://localhost:8080 by enabling and running the flood_prediction_pipeline DAG. Predictions can be made by sending a POST request to http://localhost:8000/predict with the required input features.
